Blackjack Trainer

Blackjack Trainer is a software tool designed to teach both basic strategy and card counting for blackjack players.

The trainer features a dynamic basic strategy chart that adapts to the current game state. The algorithm continuously evaluates the player's hand and the dealer's visible card, then recommends the optimal action—hit, stand, double down, or split. If certain actions become unavailable due to game rules or previous moves, the chart automatically updates to suggest the next best option.

This algorithmic approach helps users understand why each decision is optimal in any situation, improving both strategy and decision-making skills.
The project emphasizes practical learning and strategy optimization, providing a hands-on way to master blackjack fundamentals through real-time feedback.

Wave Function Collapse

This project implements the Wave Function Collapse (WFC) algorithm for procedural level generation.

The algorithm analyzes a set of input tiles and their adjacency rules, then generates new levels that respect these constraints. My implementation supports tile-based environments, ensures connectivity, and allows flexible input patterns to produce varied and playable layouts.

Additionally, the system includes a visualization of the collapsing process, making it easier to understand how local constraints propagate through the level.

This approach demonstrates WFC’s potential to automate content creation while maintaining coherent and diverse results.

Weeping Angel – Procedural Level Generation with WFC

This project builds directly on my previous Wave Function Collapse (WFC) implementation and extends it from a purely algorithmic experiment into an interactive game prototype. While the earlier project focused on the technical foundations of WFC, this proof of concept explores its practical use in gameplay-driven procedural level generation.

The goal was to evaluate whether WFC can generate organic, coherent, and replayable levels that actively support game mechanics instead of producing abstract or purely visual results. To test this, the project uses a Weeping Angel-inspired enemy concept, where enemies only move when they are not observed by the player. This mechanic places strong structural and visibility constraints on the generated environments.

Levels are generated at runtime using a custom WFC-based system with modular tiles and explicit adjacency rules. The algorithm ensures logical spatial connections, navigable layouts, and consistent transitions between rooms while still allowing randomness and variation in each playthrough.

The project intentionally prioritizes procedural systems, algorithmic design, and gameplay integration over visual fidelity or narrative depth. It demonstrates how an abstract procedural algorithm can be transformed into a concrete game system that directly influences player experience and level design.
